# HTTP Method
### GET
- 리소스 조회
- 빈 Body 전송
- POST보다 빠름 (캐싱 가능)
- 데이터를 URL에 담아 전송함
    - Path Variable (경로 변수) : 리소스 식별
    - Query Parameter (쿼리 매개변수) : 조건, 필터, 페이징 등
### POST
- 요청 데이터 처리
- 서버 상태 변경 작업에 사용함, 주로 등록에서 사용함
- 요청 정보를 Body 안에 숨겨 서버로 전송함
- 대용량 데이터 전송 적합
### PUT
- 리소스 대체
- 해당 리소스가 없으면 생성함
### PATCH
- 리소스 부분 변경
### DELETE
- 리소스 삭제
